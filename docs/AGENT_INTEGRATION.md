# Moongatracker — интеграция с агентом

Как AI-агент (OpenClaw в мессенджере, Claude Code, n8n-флоу) работает с доской.

## Принцип

Агент не имеет прямого доступа к БД. Он ходит через **REST API** по **API-токену**, а для
удобного «инструментального» доступа поверх API поднят **MCP-сервер** (`apps/mcp`).
Каждая мутация фиксируется в `Activity` — это трейс и точка отката.

## Аутентификация агента

- Сущность `ApiToken` (см. DESIGN.md): `name`, `scope`, `tokenHash`, отзыв.
- Токен передаётся в `Authorization: Bearer <token>`.
- Скоупы (минимально): `cards:read`, `cards:write`, `comments:write`.
- Токен агента **не** является пользовательской сессией — отдельный субъект в аудите
  (`actorType = agent`).

## MCP-инструменты (apps/mcp)

Тонкая обёртка над REST. Предлагаемый набор tools:

| Tool | Назначение |
|------|-----------|
| `list_boards` | список досок |
| `list_cards` | карточки (фильтр по `boardId`, `columnKey`, лейблам) |
| `get_card` | полная карточка с комментариями и историей |
| `create_card` | создать карточку (по умолчанию в `idea`) |
| `update_card` | заголовок/тело/приоритет/лейблы |
| `move_card` | сменить `columnKey` (idea→triage→backlog→in_dev→done) |
| `comment_card` | добавить комментарий (`authorType=agent`) |
| `list_activity` | история действий по карточке |

`move_card` и `update_card` — это и есть «перенос идеи в разработку» одним вызовом.

## Трейс и откат

- Любая мутация пишет `Activity { action, before, after, actorType, actorId }`.
- UI помечает действия агента (значок «agent») и показывает историю карточки.
- Эндпоинт отката восстанавливает `before` для выбранной записи `Activity`.

## Человеко-гейт

Необратимое или «дорогое» агент **предлагает**, а не делает молча:

- Перевод в `in_dev`, удаление, массовые операции → агент создаёт предложение/комментарий,
  человек подтверждает (в UI или ответом в мессенджере).
- Конкретный список «гейтнутых» действий уточняется на этапе плана.

## Сценарий «разбор идеи» (end-to-end)

```
1. Пользователь в Telegram: «идея: автотегирование отзывов по тону»
2. OpenClaw → create_card(title=..., column=idea)
3. OpenClaw разбирает, задаёт вопросы в чате, пишет резюме:
   → comment_card(...), update_card(labels=[...]), move_card(triage)
4. Решение «берём» (человеко-гейт) → move_card(backlog),
   update_card(body="мини-спека...")
5. Всё действие видно в list_activity, при ошибке — откат.
```

## Связь с n8n

n8n остаётся для **детерминированной** части (расписания, вебхуки, бэкап SQLite,
уведомления). «Думающие» шаги делегируются агенту через те же REST/MCP.
