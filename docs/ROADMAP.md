# Moongatracker — роадмап

Поэтапно, каждый этап — самостоятельная ценность. Детальный план реализации
составляется отдельно для каждой фазы.

## Phase 0 — каркас монорепо ✅ (готово)

- Nx workspace, `apps/api` (NestJS+Fastify), `apps/web` (Vite+React+shadcn), `libs/shared-types`, `libs/data-access`.
- Prisma + **PostgreSQL** (внешний), init-миграция, схема (User, Board, Column, Card), идемпотентный сид (демо-доска + задачи).
- `GET /api/boards` (доска + колонки + карточки).
- Dockerfile + docker-compose (api подключается к внешнему Postgres).
- **Готово:** `nx serve` поднимает api+web, доска с демо-данными рендерится.

## Phase 1 — MVP канбана (человек) ✅ (готово)

- CRUD карточек/колонок, drag&drop, комментарии, лейблы.
- Auth (email+пароль, JWT).
- Два view: «Идеи» / «Разработка» поверх статусов.
- Socket.IO — живое обновление.
- React Query на фронте; раздача `web` из api (один контейнер).
- **Готово:** можно вести идею→разработку руками в браузере.

## Phase 2 — агент-слой ✅ (готово)

- `ApiToken` + скоупы.
- `Activity`-трейс на все мутации + откат.
- Человеко-гейт на необратимые действия агента (перевод в `in_dev`, удаление, массовое).
- `apps/mcp` с набором tools (см. AGENT_INTEGRATION.md); подключение к Claude Code.
- **Готово:** агент создаёт и двигает карточки через MCP, действия видны в истории и обратимы.

## Phase 3 — мультитенант: орги + проекты + новая модель + sidebar + настройки

Детали: `docs/superpowers/specs/2026-06-29-multitenant-hierarchy-plan.md` → Фаза 1.

- Рефакторинг схемы: `Board → Project`, `columnKey → columnId` (FK), `priority: Int → String?` (urgent/normal/low), убрать `Label`/`CardLabel`, добавить `authorType/assigneeType` на `Card`.
- Новые модели: `Organization`, `Membership`.
- `ApiToken` переводим на `orgId`.
- Свободные колонки: создание, переименование, удаление, DnD перестановка.
- Приоритет карточки: urgent/normal/low, двухуровневая сортировка.
- Автор и исполнитель карточки (user или agent).
- Backend: JWT-auth guard, Organization CRUD, Project CRUD с org-скоупом.
- Frontend: роутер, login/register, список проектов.
- `<Sidebar />`: переключатель орг, список проектов, «+ Новый проект», ссылка «Настройки», профиль/выход.
- `/settings`: скелет с вкладками **Организация** (переименовать), **Участники** (только просмотр), **AI-агенты** (только просмотр).
- **Готово:** регистрируюсь, создаю оргу и проекты, хожу между ними через sidebar, веду канбан с приоритетами.

## Phase 4 — участники + токены агента + MCP

Детали: `docs/superpowers/specs/2026-06-29-multitenant-hierarchy-plan.md` → Фаза 2.

- Добавление участника по email, без ролей; удаление.
- `ApiToken` CRUD (создать/отозвать), токен-auth guard, `actorType=agent`.
- `Activity`-трейс на мутации агента + эндпоинт отката.
- MCP: дополнить `update_card`, `comment_card`, `list_activity`.
- `/settings`: вкладки **Участники** и **AI-агенты** становятся полностью рабочими.
- Frontend: маркер «agent» на действиях/комментариях, история карточки.
- **Готово:** добавляю людей и агента через настройки, агент рулит доской через MCP.

## Phase 5 — скриншоты на карточках

Детали: `docs/superpowers/specs/2026-06-29-multitenant-hierarchy-plan.md` → Фаза 3.

- Внешний S3-бакет через env-переменные (`S3_ENDPOINT`, `S3_BUCKET`, `S3_ACCESS_KEY`, `S3_SECRET_KEY`); presigned-upload (фронт льёт напрямую в S3).
- `Attachment` CRUD (прикрепить, список, удалить + чистка в S3).
- Frontend: загрузка файла + Ctrl+V скриншота в `CardDialog`, превью-галерея, лайтбокс, бейдж на `CardTile`.
- **Готово:** к карточке можно прикрепить скриншоты, посмотреть и удалить.

## Принципы по ходу

- YAGNI: не тащим SSO/продвинутые роли/аналитику без запроса.
- Каждый этап деплоится и проверяется на реальном сервере до следующего.
