# Лендинг moongatracker — дизайн (контент + каркас)

Дата: 2026-07-05
Статус: согласован, готов к плану реализации

## 1. Цель и контекст

Маркетинговый лендинг для **moongatracker** — облачного (SaaS) канбана, в котором
**AI-агент — полноправный участник**. Задача лендинга — привести целевого посетителя к
**регистрации** в приложении.

- **Продукт:** облачный SaaS (НЕ self-hosted — упоминаний self-host/docker/внешней БД/RAM в
  контенте быть не должно).
- **Целевое действие:** регистрация (`/register` уже существует в `apps/web`).
- **Аудитория:** разработчики, которые уже используют AI-агентов (Claude / OpenClaw). Тон —
  технический: MCP, REST, токены, трейс, откат.
- **Язык:** русский.

## 2. Нарратив

Гибрид «общая память» (A) + «агент ведёт доску» (B):
ведём ценностью «агент — участник + общая память человека и агента», а **трейс / откат /
человеко-гейт** даём как аргумент доверия. Технику (MCP / REST / токены) выносим в карточку
интеграций.

## 3. Каркас (переиспользуем из референса `~/ai-reviews-landing`)

Механизм — **вертикальный стек полноэкранных карточек** со сменой «один жест = один экран»
(GSAP ScrollTrigger `pin`+`scrub`, Lenis smooth-scroll, дискретная навигация wheel/touch/
клавиши, контент активной карточки через `[data-fade-on-scroll]`). Позиции заголовков/CTA/
блоков — как в референсе.

**Стиль — из moongatracker, НЕ придумываем:**
- Шрифт: **JetBrains Mono Variable** (`--font-heading: var(--font-mono)`).
- Primary: `oklch(0.514 0.222 16.935)` (розово-красный, hue ~17); полный набор shadcn-токенов
  на oklch из `apps/web/src/styles.css` (light + dark).
- Радиусы/эстетика — dev-tool, как в приложении.

**Карточек: 5** (референс каркаса тот же, контента больше).

## 4. Контент по карточкам

### Карточка 1 — Hero
- **H1:** «Канбан, где AI-агент — полноправный участник»
- **Подзаголовок:** «Идея живёт на одной карточке — от наброска до релиза. Агент читает,
  создаёт и двигает карточки через MCP, а ты видишь каждый его шаг и в один клик
  откатываешь.»
- **CTA (основной):** «Начать бесплатно» → регистрация.
- **CTA (ghost):** «Смотреть демо» → скролл к карточке 3.
- **Фон:** слот фоновой анимации (как в референсе) — смысл: «живой» поток доски / активности
  агента. Конкретная визуалка — из готового стиля.

### Карточка 2 — Как это работает (жизненный цикл)
- **H2:** «От идеи до релиза — на одной карточке»
- **Шаги (4, появляются по очереди):**
  1. **Кидаешь идею** — в чат агенту (Telegram / Claude) или прямо на доску.
  2. **Агент разбирает** — вопросы, ресёрч, оценка, лейблы → двигает в «Разбор».
  3. **«Берём»** — карточка уходит в разработку, агент дописывает мини-спеку в тело.
  4. **Ведётся до «Готово»** — обсуждение и вся история остаются на карточке.
- **Подпись:** «Один путь `idea → разбор → backlog → in dev → done`, два вида: «Идеи» и
  «Разработка».»

### Карточка 3 — Интерактивное демо (аналог демо-блока референса)
- **H2:** «Посмотри, как агент ведёт карточку»
- **Механика:** редактируемая мини-карточка идеи + кнопки-примеры. Кнопка **«Отдать агенту»**
  → состояние «агент думает…» → эффектом печати появляется **комментарий агента**, вешаются
  **лейблы**, карточка **переезжает `idea → разбор`**. Внизу строка Activity:
  «агент переместил карточку: idea → разбор» + кнопка **«Откатить»** (возвращает состояние —
  демонстрирует трейс и обратимость).
- **Примеры идей:** «Тёмная тема для настроек», «Экспорт доски в Markdown»,
  «Вебхук в Telegram на смену статуса».
- **Бэкенд демо:** без внешнего LLM — детерминированная заготовка ответа/лейблов по идее
  (как эвристика в референсе), без логирования введённого текста.

### Карточка 4 — Хаб интеграций (лучи, как в референсе)
- **H2:** «Твой агент работает с доской откуда угодно»
- **Подзаголовок:** «MCP-тулы и REST — из Claude, Telegram или своих скриптов. Токены со
  скоупом, живое обновление, каждый шаг в трейсе.»
- **Центр:** moongatracker. **Узлы (двунаправленные лучи):** Claude · OpenClaw (Telegram) ·
  MCP-сервер · REST API · n8n · Socket.IO (realtime) · API-токены.

### Карточка 5 — Тарифы / финальный CTA
- **H2:** «Начни бесплатно»
- **3 тарифа (на старте без цифр — цены проставляются позже):**
  - **Free** — 1 доска, 1 агент-токен, история активности 7 дней. «Для одного проекта».
  - **Pro** — безлимит досок и токенов, полный трейс и откат, приоритетный realtime.
  - **Team** — роли, участники, поддержка (SSO — позже).
- **Финальный CTA:** «Создать аккаунт» → регистрация.

## 5. Открытые вопросы / дефолты

- **Цены тарифов** — на старте не показываем цифры (дефолт). Проставить перед запуском.
- **Куда ведут CTA** — на существующий `/register` приложения moongatracker.
- **Размещение лендинга (решается в плане реализации):** отдельный проект, зеркалящий стек
  референса (Next.js 16 + GSAP + Lenis), стилизованный токенами moongatracker + JetBrains
  Mono; CTA ведут на `/register` приложения. Подтвердить в плане.

редизайн визуального стиля (берём готовый из `apps/web`).
